    Function builderIf

    • builderIf<T extends Record<string, TPlugin>>(ctx?: T): TBuilder<T>

      General documentation

      Initializes the "IF" context.

      The modularity of builderIf allows you to flexibly load checks (plugins) into the context and interact with internal service context variables.

      Type Parameters

      • T extends Record<string, TPlugin>

        Current downloaded plugins via the plugin function

      Parameters

      • ctx: T = ...

        The context with which the current builder is initialized. It must be an object where the keys are string names of plugins, and the values are functions of type TPluginBody. This parameter is used internally. For now, it only supports functions without parameters (to be improved in future versions).

      Returns TBuilder<T>

      An extended context builder with the following:

      • plugin — A method to expand the context by registering a new plugin. Takes the plugin name and the plugin function.
      • s — A service function to call a plugin with arguments.
      • Any other previously added plugins.

      Throws

      Error If the requested plugin does not exist in the s function.

      Throws

      Error If the plugin function receives a value that is not a function.

      Guides

      1. To type this in the plugin function used by s, use the TContext type.
      const builder = builderIf().plugin("fnFirst", function(this: TContext) {
        // you can use `this.tmp` inside
      });
      1. Use the TPlugin type to define the plugin. It accepts a tuple of argument types as a generic.
      const myPlugin: TPlugin<[string, number]> = (str, num) => {
        return str.length === num;
      };
      const x = builderIf().plugin('myPlugin', myPlugin);

      This utility type automatically infers the return type as boolean and types the plugin parameters accordingly.

      1. Use the bIf wrapper instead of using builderIf

      Example

      builderIf().plugin(
        'compareValues',
        function(this: TContext, a: string, b: string) {
          return a === b;
        }
      );

      Config

      The configuration applies to the entire facility and is permanent. You can find the configuration type in TConfig The configuration is optional and has default values. Please read this documentation before changing anything in it. Description of the configuration keys:

      • autoAddContext - If true is set, then EACH added plugin can interact with the context, this parameter eliminates the need to use s' if you need context. Activate it if you don't want to bother with the s` and brevity and conciseness are important to you.
